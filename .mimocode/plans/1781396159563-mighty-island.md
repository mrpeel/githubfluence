# Plan: Fix Live Tab Not Appearing on Edit Page

## Root Cause

`addLiveTab()` in `src/content/github-page.ts:122` is called synchronously by `setupEditPage()` in `src/content/index.ts:76` when the content script runs at `document_end`. GitHub's edit page is a React SPA — the `ul[aria-label="Edit mode"]` segmented control hasn't been rendered yet at that point. `addLiveTab()` returns `null` because `document.querySelector('ul[aria-label="Edit mode"]')` finds nothing, and there is no retry/polling logic.

## Fix

Add a `waitForElement` helper that uses a `MutationObserver` to wait for the target selector to appear in the DOM, with a timeout fallback. Use this in `setupEditPage()` instead of calling `addLiveTab()` directly.

### Changes

**`src/content/github-page.ts`** — Add `waitForElement()` helper:
```typescript
function waitForElement(selector: string, timeout = 5000): Promise<Element | null> {
  const existing = document.querySelector(selector);
  if (existing) return Promise.resolve(existing);

  return new Promise((resolve) => {
    const observer = new MutationObserver(() => {
      const el = document.querySelector(selector);
      if (el) { observer.disconnect(); resolve(el); }
    });
    observer.observe(document.body, { childList: true, subtree: true });
    setTimeout(() => { observer.disconnect(); resolve(null); }, timeout);
  });
}
```

**`src/content/index.ts`** — Change `setupEditPage()` to be async and await the element:
```typescript
async function setupEditPage() {
  await waitForElement('ul[aria-label="Edit mode"]');
  const tab = addLiveTab(() => { ... });
  // rest of existing tab-click handler logic
}
```

Add `waitForElement` to the import from `github-page.ts`.

### Files to modify
- `src/content/github-page.ts` — add `waitForElement` function, export it
- `src/content/index.ts` — make `setupEditPage()` async, await `waitForElement` before `addLiveTab()`
